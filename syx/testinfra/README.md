# netns

Isolated Linux network namespaces for Firecracker microVM(s).

## Architecture

Each node is a Linux network namespace containing a Firecracker microVM and
per-node systemd services (`syxvm`, `syxgw`).

The host bridge `br-fc` connects all nodes. Each node has a veth pair: one end
attached to the bridge, the other inside the namespace. Inside the namespace, a
tap device connects to the microVM via virtio-net. Outbound traffic from the
microVM is masqueraded (NAT) at the namespace boundary via the veth interface.

Addresses:

```
bridge:       10.0.0.1/24    fd10::1/64
---
node_a veth0: 10.0.0.2/24    fd10::2/64
node_a tap0:  172.16.0.0/31  fdfc:a::0/127
vm0    eth0:  172.16.0.1/31  fdfc:a::1/127
---
node_b veth0: 10.0.0.3/24    fd10::3/64
node_b tap0:  172.16.0.0/31  fdfc:b::0/127
vm0    eth0:  172.16.0.1/31  fdfc:b::1/127
```

Default routes inside the VM are configured by `setup_vm_network.yml` after boot.
The rootfs auto-configures the IPv4 address from the MAC address (`fcnet-setup.sh`),
but does not set default routes or IPv6.

Link-local IPv6 addresses (`fe80::/64`) are auto-generated by the kernel from
the MAC address on all interfaces.

## Setup

Run host and all nodes in order:

```sh
terramate script run -- up
```

Or individually:

```sh
terramate script run -C syx/testinfra/host -- up
terramate script run -C syx/testinfra/node/node_a -- up
```

Host `up` runs:
1. `setup_host.yml` - one-time host setup (idempotent)
1. `create_bridge.yml` - create bridge (survives until reboot)

Node `up` runs:
1. `create_netns.yml` - namespace, veth, tap, nftables, syxvm/syxgw services
1. `start.sh` - start Firecracker inside namespace
1. `setup_vm_network.yml` - configure default routes and IPv6 inside microVM

## SSH into microVM

VM IP (`172.16.0.1`) is not reachable from the host's network namespace;
it only exists inside the node namespace.
SSH through nc running inside the namespace:

```sh
ssh -i syx/testinfra/testkey \
    -o ProxyCommand='sudo ip netns exec node_a nc -6 %h %p' \
    root@fdfc:a::1
```

## Teardown

Teardown runs in reverse order (nodes first, then host):

```sh
terramate script run --reverse -- down
```

Note that the namespace, Firecracker, and bridge are ephemeral.
They are lost on teardown or reboot.

## Services

Each node has two systemd service definitions (not yet implemented; both run
`sleep infinity` as a placeholder):

- `syxvm@node_a.service` - VM management daemon
- `syxgw@node_a.service` - eBPF gateway

Service templates are installed by `setup_host.yml` and instantiated per node
by `create_netns.yml`.
